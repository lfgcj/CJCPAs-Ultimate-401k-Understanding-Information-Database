# 401(k) Plan Mergers and Acquisitions

## Overview

Corporate mergers, acquisitions, and other business combinations create complex challenges for 401(k) plan management. Proper handling of plan mergers, spin-offs, and asset transfers is essential to maintain qualified plan status and protect participant benefits. This comprehensive guide covers all aspects of 401(k) plans in M&A transactions.

## M&A Fundamentals for 401(k) Plans

### Impact of Corporate Transactions

| Transaction Type | Impact on Plans |
|-----------------|-----------------|
| Stock acquisition | Plans may remain separate or merge |
| Asset acquisition | Employees become new employer's |
| Merger | Companies combine, plans may merge |
| Spin-off | Division becomes separate company |
| Carve-out | Subsidiary sold or goes public |

### Key Decision Points

```
M&A Decision Tree:

Corporate Transaction Announced
           ↓
Identify Affected Plans
           ↓
Determine Transaction Structure
           ↓
Options:
├── Merge plans
├── Maintain separate plans
├── Terminate acquired plan
└── Spin-off plan assets

Consider:
- Business objectives
- Plan design differences
- Participant impact
- Administrative efficiency
- Cost implications
- Compliance requirements
```

### Parties Involved

| Party | Role |
|-------|------|
| Acquiring company | Decision-maker for combined entity |
| Target company | Provides plan information |
| Plan administrators | Execute plan changes |
| Recordkeepers | System integration |
| Legal counsel | Compliance oversight |
| Actuaries/TPAs | Testing, compliance |
| Auditors | Audit considerations |

## Types of Plan Transactions

### Plan Merger

Two or more plans combine into one:

| Element | Description |
|---------|-------------|
| Definition | Assets and liabilities of one plan transferred to another |
| Surviving plan | Plan that continues after merger |
| Merged plan | Plan that terminates upon merger |
| Effective date | Date merger is complete |

### Plan Spin-Off

One plan divides into two or more:

| Element | Description |
|---------|-------------|
| Definition | Portion of plan assets transferred to new plan |
| Original plan | Plan from which assets transfer |
| New plan | Plan receiving transferred assets |
| Transfer population | Participants whose assets transfer |

### Asset Transfer

Transfer of assets between plans:

| Type | Description |
|------|-------------|
| Plan-to-plan transfer | Assets move between qualified plans |
| Participant transfer | Individual transfers with participants |
| Bulk transfer | Group of participants transferred |

### Plan Termination

Complete termination of plan:

| Element | Description |
|---------|-------------|
| Full vesting | All participants become 100% vested |
| Distribution | All assets distributed to participants |
| IRS filing | May need to request determination letter |
| DOL filing | Final Form 5500 |

## Plan Merger Requirements

### IRC Section 414(l) Requirements

Plans may merge only if immediately after:

| Requirement | Description |
|-------------|-------------|
| Benefit preservation | Each participant entitled to benefit at least equal to pre-merger benefit |
| Vesting preservation | Vesting cannot be reduced |
| Protected benefits | Certain benefits must be preserved |

### Pre-Merger Analysis

| Analysis | Purpose |
|----------|---------|
| Plan comparison | Identify differences in plan terms |
| Benefit comparison | Ensure no benefit reduction |
| Vesting comparison | Verify vesting protection |
| Investment mapping | Plan investment transitions |
| Fee analysis | Evaluate fee impact |

### Plan Comparison Checklist

```
Plan Comparison Analysis:

Feature                    Plan A        Plan B        Decision
─────────────────────────────────────────────────────────────────
Eligibility age           21            18            Use 18
Service requirement       1 year        6 months      Use 6 months
Entry dates               Semi-annual   Quarterly     Use quarterly
Match formula             50% to 6%     100% to 4%    Evaluate
Vesting schedule          3-yr cliff    6-yr graded   Must protect
Loan provisions           Yes           No            Add loans
Hardship                  Yes           Yes           Maintain
Auto-enrollment           No            Yes           Add auto
```

### Protected Benefits

Benefits that must be preserved:

| Benefit | Protection |
|---------|------------|
| Accrued benefit | Cannot reduce vested amount |
| Early retirement | Subsidized early retirement |
| Optional forms | Payment forms available |
| Vesting | Cannot reduce vesting |

### Merger Documentation

| Document | Purpose |
|----------|---------|
| Board resolution | Authorize merger |
| Plan amendment | Amend surviving plan |
| Merger agreement | Document merger terms |
| Participant notice | Inform participants |
| Summary of Material Modifications | Required disclosure |

## Plan Spin-Offs

### When Spin-Offs Occur

| Situation | Description |
|-----------|-------------|
| Corporate divestiture | Selling subsidiary |
| Business unit sale | Transferring division |
| Corporate spin-off | Creating separate public company |
| Partial termination | Transferring terminated group |

### IRC Requirements for Spin-Offs

| Requirement | Description |
|-------------|-------------|
| Section 414(l) | Benefit preservation |
| Allocation of assets | Based on account balances |
| Vesting protection | Maintain vesting service |
| Nondiscrimination | Cannot discriminate in selection |

### Spin-Off Asset Allocation

Assets transferred must equal:

```
Spin-Off Asset Calculation:

For each transferring participant:
Transfer Amount = Participant's Total Account Balance

Verification:
- Total assets transferred = Sum of individual accounts
- Remaining plan assets = Original assets - Transferred assets
- All accounts properly allocated
```

### Spin-Off Process

```
Spin-Off Timeline:

Month 1-2: Planning
- Identify transferring participants
- Determine spin-off date
- Draft plan documents

Month 3-4: Documentation
- Execute plan amendments
- Prepare participant notices
- Establish new plan

Month 5-6: Implementation
- Transfer assets
- Map investments
- Update records

Month 7+: Post-Spin-Off
- Reconcile transfers
- File Form 5500s
- Address questions
```

## Asset and Liability Transfers

### Transfer Requirements

| Requirement | Description |
|-------------|-------------|
| Trustee-to-trustee | Direct transfer between trustees |
| Valuation | Assets valued at transfer date |
| Documentation | Transfer agreement required |
| Participant consent | Generally not required for merger |

### Types of Transfers

#### Direct Trustee-to-Trustee

| Characteristic | Description |
|----------------|-------------|
| Mechanism | Assets move directly between trusts |
| Tax treatment | No taxable event |
| Documentation | Transfer agreement |
| Timing | Can be same-day or extended |

#### In-Kind vs. Cash Transfers

| Type | Advantages | Disadvantages |
|------|------------|---------------|
| In-kind | No market timing, lower costs | May require investment mapping |
| Cash | Clean transfer, simple | Market timing risk, transaction costs |

### Investment Mapping

When investments differ between plans:

| Mapping Approach | Description |
|------------------|-------------|
| Like-to-like | Match similar investment options |
| QDIA | Transfer to qualified default |
| Participant election | Allow participants to choose |
| Hybrid | Combination of approaches |

### Sample Investment Mapping

```
INVESTMENT MAPPING TABLE

Acquired Plan Fund         → Acquiring Plan Fund
───────────────────────────────────────────────────
Large Cap Growth A        → Large Cap Growth X
Large Cap Value B         → Large Cap Value Y
Bond Fund C               → Bond Index Z
Stable Value D            → Stable Value W
Target Date 2030          → Target Date 2030
Target Date 2040          → Target Date 2040
Company Stock             → Self-Directed Brokerage*

*Special handling required for company stock
```

## Due Diligence Process

### Pre-Transaction Due Diligence

| Area | Review Items |
|------|--------------|
| Plan documents | Current plan, amendments, SPD |
| Compliance | Testing results, corrections |
| Operations | Contributions, distributions, loans |
| Financial | Audit reports, Form 5500s |
| Vendors | Service agreements, fees |
| Litigation | Any plan-related litigation |

### Due Diligence Checklist

```
401(k) Plan Due Diligence:

Plan Documents:
☐ Current plan document
☐ All amendments
☐ Summary Plan Description
☐ Trust agreement
☐ IRS determination letter

Compliance:
☐ Most recent ADP/ACP testing
☐ Coverage testing (410(b))
☐ Top-heavy determination
☐ 415 limit testing
☐ Any correction submissions

Operations:
☐ Contribution procedures
☐ Distribution procedures
☐ Loan procedures
☐ Participant data accuracy

Financial:
☐ Last 3 audit reports
☐ Last 3 Form 5500s
☐ Investment performance
☐ Fee schedule

Vendors:
☐ Recordkeeper agreement
☐ TPA agreement
☐ Investment advisory agreement
☐ Audit engagement letter
```

### Common Due Diligence Findings

| Finding | Risk Level | Resolution |
|---------|------------|------------|
| Late contributions | High | Calculate and deposit lost earnings |
| Failed testing | High | Correct before close |
| Document errors | Medium | Amend documents |
| Missing participants | Medium | Conduct search |
| Fee issues | Medium | Evaluate and address |
| Operational errors | Varies | Correct under EPCRS |

### Representations and Warranties

Typical plan-related reps and warranties:

| Representation | Description |
|----------------|-------------|
| Qualified status | Plan is and has been qualified |
| ERISA compliance | Plan complies with ERISA |
| No prohibited transactions | No uncorrected PTs |
| Proper administration | Plan properly administered |
| Accurate information | Information provided is accurate |

## Transition Planning

### Transition Timeline

```
M&A Transition Timeline:

Pre-Closing (Months -6 to -1):
- Due diligence complete
- Plan decisions made
- Transition plan developed
- Vendor selection (if changing)
- Document preparation

Closing (Month 0):
- Transaction closes
- Employment transfers
- Determine plan effective dates

Post-Closing (Months 1-6):
- Execute plan amendments
- Begin participant communications
- Process eligibility changes
- Implement plan changes

Integration (Months 6-12):
- Complete system conversions
- Final asset transfers
- Reconciliation
- Ongoing operations
```

### Service Provider Considerations

| Consideration | Options |
|---------------|---------|
| Recordkeeper | Keep separate, convert, or use new |
| TPA | Consolidate or maintain separate |
| Auditor | May need single auditor |
| Investment platform | Map or maintain investments |

### System Conversion Planning

| Phase | Activities |
|-------|------------|
| Planning | Define requirements, timeline |
| Data mapping | Map data between systems |
| Testing | Test conversion accuracy |
| Parallel | Run parallel systems |
| Conversion | Execute conversion |
| Validation | Verify data accuracy |

## Participant Communications

### Required Notices

| Notice | When Required |
|--------|---------------|
| Summary of Material Modifications | Plan changes |
| Blackout notice | If rights suspended 3+ days |
| QDIA notice | If default investment changes |
| Safe harbor notice | If safe harbor provisions change |
| 404(a)-5 fee disclosure | If investment/fee changes |

### Communication Timeline

| Timing | Communication |
|--------|---------------|
| 60+ days before | Intent to change plans |
| 45-30 days before | Blackout notice (if applicable) |
| 30 days before | QDIA notice (if applicable) |
| At change | SMM, enrollment materials |
| After change | Confirmation, new account info |

### Blackout Notice Requirements

When required (rights suspended 3+ days):

| Element | Requirement |
|---------|-------------|
| Timing | 30-60 days before blackout |
| Content | Reason, dates, affected rights |
| Method | Written notice |
| Delivery | To affected participants |

### Sample Communication Plan

```
PARTICIPANT COMMUNICATION PLAN

Phase 1: Announcement (60 days before)
- Company announcement of merger
- FAQ document
- Website updates

Phase 2: Plan Changes (45 days before)
- Blackout notice (if applicable)
- Summary of plan changes
- New enrollment materials
- Investment mapping information

Phase 3: Conversion (30 days before)
- Detailed timeline
- Action items for participants
- Contact information for questions
- Reminder communications

Phase 4: Post-Conversion
- Welcome to new plan
- Account access instructions
- Confirmation of elections
- Ongoing education
```

## Compliance Considerations

### Controlled Group Issues

| Consideration | Impact |
|---------------|--------|
| New controlled group | May affect coverage testing |
| Separate plans | May need to aggregate for testing |
| HCE determination | May change HCE status |
| Top-heavy | May affect determination |

### Testing Considerations

| Test | M&A Impact |
|------|------------|
| Coverage (410(b)) | New employee population |
| ADP/ACP | Combined testing population |
| Top-heavy | New key employee determination |
| 415 limits | Combined employer contributions |

### Plan Year Considerations

| Situation | Options |
|-----------|---------|
| Different plan years | Short plan year may be needed |
| Maintaining years | Keep existing plan years initially |
| Conforming years | Align plan years over time |

### Form 5500 Filing

| Situation | Filing Requirement |
|-----------|-------------------|
| Merged plan | Final 5500 for terminated plan |
| Surviving plan | Report merged assets |
| Spin-off | Both plans file |
| Short year | File for short period |

## Common Issues and Solutions

### Issue: Different Plan Designs

| Challenge | Solution |
|-----------|----------|
| Different match formulas | Harmonize over time or maintain |
| Different vesting | Cannot reduce vesting |
| Different eligibility | Use most favorable |
| Different investments | Map or maintain |

### Issue: Legacy Benefits

| Challenge | Solution |
|-----------|----------|
| Protected benefits | Must preserve |
| Early retirement | Document and maintain |
| Payment forms | Continue to offer |
| Special provisions | Amend to preserve |

### Issue: System Integration

| Challenge | Solution |
|-----------|----------|
| Different recordkeepers | Convert or maintain separately |
| Data differences | Map and validate data |
| Historical records | Transfer and preserve |
| Investment history | Maintain for participants |

### Issue: Timing Pressures

| Challenge | Solution |
|-----------|----------|
| Tight closing timeline | Prioritize critical items |
| Complex integration | Phase implementation |
| Participant impact | Clear communication |
| Compliance deadlines | Plan ahead |

### Issue: Multiple Plans

| Challenge | Solution |
|-----------|----------|
| Too many plans | Develop consolidation strategy |
| Testing complexity | Consider aggregation |
| Administrative burden | Streamline over time |
| Fee efficiency | Consolidate for scale |
